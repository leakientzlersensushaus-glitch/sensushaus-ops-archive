# Facilities And Support Role Summary

This note summarizes the practical difference between ordinary facilities control, contractor support, architecture review, and continuity oversight in protected technical environments.

## Architecture Review

Architecture review personnel can:

- inspect environment definitions
- compare review snapshots
- query room-event history

Architecture review personnel cannot:

- trigger incident mode
- alter room occupancy flags
- clear or restore room-control sequences

## Facilities Environment Administration

Facilities administration can:

- manage routine environmental settings
- run scheduled tests
- review protected-area room state

Facilities administration normally acts through standard operational workflows and local change handling.

## Compliance Review

Compliance review can:

- review access and event records
- request supporting material
- retain records for later review

Compliance review cannot directly control protected room state.

## Vendor Diagnostic Support

External vendor support can:

- perform limited diagnostics
- assist under ticketed supervision

Vendor support cannot initiate suppression, room isolation, or incident-mode workflow in protected spaces.

## Continuity Oversight

Continuity oversight exists for designated review, escalation, and continuity-critical routing under documented governance arrangements.

Where explicitly permitted, this scope may interact with protected-environment control paths outside ordinary daily workflows.
